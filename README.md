# Сервис поиска и бронирования аудиторий МатМех УрФУ

## Цель проекта

Разработать web-приложение для студентов и преподавателей МатМех УрФУ, предназначенное для поиска и бронирования свободных аудиторий.

## Описание проекта

На МатМехе нередко возникают ситуации, когда необходимые аудитории уже заняты, из-за чего преподавателям и студентам приходится искать свободный кабинет для проведения занятий, консультаций или самостоятельной работы.

Разрабатываемый сервис позволит:

* просматривать занятость аудиторий в режиме реального времени;
* находить свободные аудитории по заданным параметрам;
* бронировать аудитории на определённое время;
* наглядно отображать статус аудиторий на плане этажей;

## Основной функционал

### Просмотр занятости аудиторий

- Отображение аудиторий на плане этажей.
- Визуальная индикация статуса:
  - 🟢 Свободна.
  - 🔘 Занята.

### Бронирование аудиторий

* Выбор аудитории.
* Выбор даты и времени.
* Создание бронирования.

### Поиск и фильтрация

Поддерживаются фильтры по:

* дате;
* времени;
* этажу;
* другим характеристикам аудитории.

## Требования к интерфейсу

### Десктопная версия

* Поддержка ПК.
* План 5–6 этажей в виде схемы сверху.
* Цветовая индикация занятости аудиторий.
* Панель фильтров и поиска.
* Информация об аудитории при выборе.

### Мобильная версия

Отдельный адаптивный интерфейс:

* быстрый поиск свободных аудиторий;
* карточки аудиторий с основной информацией;
* возможность бронирования в несколько нажатий;
* адаптация под вертикальную ориентацию экрана.

## Дополнительные требования

### Аутентификация

Система должна поддерживать авторизацию студентов:

* идентификации пользователей;
* создания и управления бронированиями;

### Безопасность

* Защита пользовательских данных.
* Проверка прав доступа при выполнении операций бронирования.
* Логирование действий пользователей.

## Технологические требования

### Backend

* REST API для взаимодействия клиента и сервера.
* База данных для хранения информации об аудиториях, пользователях и бронированиях.
* Система аутентификации и авторизации.

### Frontend

* Адаптивный пользовательский интерфейс.
* Поддержка мобильных и десктопных устройств.
* Интерактивная карта этажей.
